# PRD: 盘前投资执行台 v3

## 1. 产品定位（核心修正）

**本页面的定位是"盘前投资执行台"（Pre-Market Execution Console），不是"投资看板"（Investment Dashboard）。**

| | 看板 | 执行台 |
|------|------|------|
| 回答的问题 | "发生了什么？" | "我今天要不要行动？怎么行动？" |
| 信息组织 | 按类型聚合（行情、新闻、告警） | 按决策需求组织（做什么、为什么、条件满足吗） |
| 用户退出时的状态 | 知道了信息 | 知道了今天的具体操作清单，完成了执行前校验 |
| 核心产出 | 数据展示 | **今日操作确认单** |

**用户决策链**：

```
看一眼健康灯（5秒）
    ↓
看今日操作确认单（30秒）—— 知道今天具体做哪几笔
    ↓
展开校验条件（按需）—— 确认每笔是否安全可执行
    ↓
执行或调整计划
```

---

## 2. 背景与问题

**问题 1：信息碎片化 — 任务与目标链路断裂**

执行看板是平铺的任务列表，用户看不到任务与远期目标的逻辑关系。当外部环境变化时，无法判断今天的任务要不要调整。

**问题 2：外部数据不可追溯**

外部事件缺少来源链接、发布时间、可信等级。投资决策依赖的数据不可追溯。

**问题 3：信息聚合而非执行导向**

页面告诉用户"应该关注什么"，但没回答"为什么现在做""做之前还要检查什么""条件变了怎么改"。

**v2 demo 评审中确认的核心缺陷**（15 条反馈归纳）：
- 任务没有分层（必须做/条件做/仅监控混在一起）
- 缺少今日操作确认单——用户看完不知道到底做哪几笔
- 外部数据没有来源链接和可信等级
- 宏观信号只有方向箭头，没有结构化影响分析
- 交易金额没有估算价格口径
- 仓位比例没有标注分母
- 标的代码不统一，存在双市场误操作风险
- 任务没有执行前校验清单
- LLM 摘要缺少引用依据和置信度
- 缺少当前持仓快照
- 告警没有关联任务和处理状态
- 事实/解读/动作建议三层混在一起

---

## 3. 产品目标

**开盘前 5 分钟，让用户安全地知道今天该不该做、做什么、怎么做。**

### 3.1 核心设计原则

- **事实 / 解读 / 动作建议三层分离**：用户始终知道哪部分是原始数据、哪部分是判断、哪部分是建议
- **数据可追溯**：外部数据标注来源链接、发布时间、更新时间、可信等级
- **执行前必校验**：每个交易任务通过校验清单
- **操作确认单是最终产出**：用户退出页面时，手里有一张明确的今日操作清单
- **不展示总资产**：避免数字波动影响情绪
- **颜色纪律**：红=必须处理/违规、黄=需确认、蓝=信息提醒、绿=已完成/利好、灰=无动作

### 3.2 排除项

- 总资产数字
- D 档美团 RSU
- 因果推理分析（盘后内容）
- WBS 进度条（月度内容）
- 电子/装饰性图标
- 历史图表/趋势图

## 4. 信息架构

```
┌─────────────────────────────────────────────────┐
│ ① 决策摘要（顶栏）                                  │
│    健康灯 + 今日必须执行 N 项 + 待确认 N 项 + 仅监控 N 项  │
├─────────────────────────────────────────────────┤
│ ② 今日操作确认单 【核心产出】                         │
│    一张表：状态 | 标的/代码/市场 | 方向 | 数量 | 估金额     │
│    每行：可执行 / 待确认 / 仅监控                     │
├─────────────────────────────────────────────────┤
│ ③ 任务执行详情（确认单的展开）                        │
│    执行前校验清单 + 条件 + 依赖 + 外部变量 + 风险提示     │
├─────────────────────────────────────────────────┤
│ ④ 当前持仓快照 【新增】                             │
│    标的/代码/市场 | 持仓量 | 成本 | 现价 | 市值 | 仓位 | 盈亏  │
├─────────────────────────────────────────────────┤
│ ⑤ 外部环境监控（微观→宏观）                         │
│    每条信号：事实层 → 解读层 → 动作层                │
│    来源链接 + 可信等级 + 发布时间                    │
├─────────────────────────────────────────────────┤
│ ⑥ 活跃告警（可折叠）                               │
│    每条告警：关联任务 + 处理状态 + 下一动作            │
└─────────────────────────────────────────────────┘
```

## 5. 各区块详细定义

### 5.1 决策摘要（顶栏）

**5 秒内获得今日行动锚点。**

| 元素 | 内容 | 来源 |
|------|------|------|
| 健康灯 | 红/黄/绿 | 综合告警+违规+止损 |
| 今日必须执行 | N 项 | 所有条件满足、无依赖阻塞的任务 |
| 今日待确认 | N 项 | 有外部变量/依赖待确认的任务 |
| 今日仅监控 | N 项 | 价格触发/条件未满足的任务 |
| critical 告警数 | N 条 | alerts 表 |

**健康灯规则**：

| 灯 | 条件 |
|----|------|
| 红 | 存在 critical 级别告警 OR 违规截止≤3天 OR 止损已触发 |
| 黄 | 存在 warning 告警 OR 有违规 OR 有今日任务 OR 冷静期到期 |
| 绿 | 以上皆无 |

### 5.2 今日操作确认单（核心新增）

**这是整个页面的核心产出。** 用户看完确认单就应该知道今天具体做哪几笔。

#### 5.2.1 表格结构

| 列 | 说明 | 示例 |
|------|------|------|
| 状态 | 可执行/待确认/仅监控 | 🟢 可执行 |
| 标的 | 名称 + 代码 + 市场 | 中证红利 ETF 515180.SH |
| 方向 | 买入/卖出/调整/暂停 | 买入 |
| 数量 | 股数/份额 | 约 30,000 份 |
| 估金额 | 估价格 × 数量 + 价格日期 | ¥48,000（估¥1.60，5/29 收盘） |
| 原因 | 一句话 + 所属目标 | 防御底仓建仓 · B仓ETF建仓 |
| 条件 | 执行前需满足的条件 | 无 |
| 执行后 | 仓位/目标变化 | B档ETF 从 3.7%→5.5% |

#### 5.2.2 状态判定规则

| 状态 | 条件 |
|------|------|
| 🟢 可执行 | 所有条件满足 + 无依赖阻塞 + 无外部变量待确认 + 不在冷静期 |
| 🟡 待确认 | 有外部变量待确认 OR 依赖任务未完成 |
| 🔵 仅监控 | 价格触发型 OR 未来日期 OR 条件尚未满足 |

#### 5.2.3 示例（5/30）

```
今日操作确认单

🟢 可执行（1项）
  买入 | 中证红利 ETF 515180.SH | 约 30,000 份 | 估 ¥48,000（估 ¥1.60，5/29 收盘）
  原因：B-P1-02 防御底仓一次性建仓
  条件：无
  执行后：B 档 ETF 从 3.7%→5.5%（占总组合 ABC）

🟡 待确认（2项）
  卖出 | 隆基绿能 601012.SH | 3,600 股 | 估 ¥54,144（估 ¥15.04，5/29 收盘）
  原因：C-003 清仓 · C仓位重组
  待确认：① 是否有停牌/重大公告 ② 开盘价偏离估价值是否 >±3%
  依赖：理想清仓(C-002)结果可能影响资金路由

  暂停/降低 | 港股科技 ETF 513010.SH | 本周定投
  原因：B-P2-01 已超买约 2.2 周额度，需回归定投节奏
  待确认：本周应投金额需重新计算

🔵 仅监控（3项）
  卖出 | 南山铝业 600219.SH | 第二档 24,600 股 | 触发价 ≥¥5.81（首批 6,150 股）
  当前价 ¥5.50 未触发
  卖出 | 理想汽车 02015.HK | 2,400 股 | 等待 Q1 财报
  买入 | 中证A500 563360.SH | 本周定投 ¥13,394 | PE 百分位 97%（条件：≤60%）
```

### 5.3 任务执行详情（确认单的展开）

确认单的每一行可点击展开，显示完整执行链路和校验清单。

#### 5.3.1 展开后结构

```
隆基绿能清仓 3,600 股 601012.SH · C-003

▸ 执行前校验
  □ 标的无停牌                      — 系统自动
  □ 无涨跌停限制                    — 系统自动
  □ 当日无重大未消化公告              — 人工+系统（最近：5/22 股东大会 ✓已消化）
  □ 开盘价在估价值 ±3% 范围内         — 系统自动
  □ 持仓数量充足（当前持有 3,600 股 ✓） — 系统自动
  □ 最小交易单位满足（100 股 ✓）      — 系统自动
  □ 冷静期已过（卖出冷静期 3 天 ✓）    — 系统自动

▸ 目标链路
  合规整改（主题集中 84%→35%，截止 6/22）
  → C仓位重组（清仓低分 thesis 标的）
  → C-003 隆基清仓

▸ 外部变量
  5/22 股东大会：BC 电池 28.13% 世界纪录 + 集采第一    [事实]
  组件集采价格 ↘（N型跌破 ¥0.75/W）                    [事实]
  → 技术面偏多，价格面偏空，综合不改变清仓计划            [解读]
  → C-003 按计划执行                                    [动作]

▸ 风险提示
  · 清仓后 C 档总仓位从 48%→约 43%，仍超 30% 合规线
  · 理想清仓(C-002)尚未执行，同步推进可能加速 C 档降仓
  · 资金回笼后路由：70%→B 档 ETF 建仓 / 30%→A 档现金

▸ 执行历史
  本次为 C-003 首次执行
```

#### 5.3.2 校验清单字段定义

| 校验项 | 类型 | 数据来源 |
|------|------|------|
| 标的是否停牌 | 系统自动 | quotes 表或交易状态 API |
| 是否涨跌停 | 系统自动 | 开盘价 vs 涨跌停价 |
| 是否有未消化公告 | 人工+系统 | 近 3 日公司公告 |
| 开盘价偏离估值 | 系统自动 | >±3% 标黄 |
| 持仓数量是否充足 | 系统自动 | holdings 表对比 |
| 最小交易单位 | 系统自动 | A 股 100 股，港股依标的 |
| 冷静期是否满足 | 系统自动 | decisions 表计算 |

### 5.4 当前持仓快照（新增）

**执行前确认"我实际持有什么"，防止任务数据与账户不同步。**

| 列 | 来源 | 示例 |
|------|------|------|
| 标的 | instruments | 比亚迪 002594.SZ |
| 市场 | instruments | A 股深圳 |
| 持仓量 | holdings.shares | 400 股 |
| 成本价 | holdings.cost_price | ¥218.50 |
| 最新价 | quotes.close（最近交易日） | ¥188.35 |
| 持仓市值 | 持仓量 × 最新价 | ¥75,340 |
| 浮盈亏% | (最新价-成本价)/成本价 | -13.79% |
| 占 C 档仓位% | 市值 / C档总市值 | 16.4% |
| 占总组合(ABC)% | 市值 / ABC总市值 | 6.4% |
| 目标仓位% | capital.yaml + 策略 | 5% |
| 偏离 | 实际 - 目标 | +11.4pp |
| 档位 | instruments.tranche | C |
| 策略状态 | thesis/execution | 观察仓 / 待减仓 |

**仓位口径规范**：所有比例必须标注分母。格式为 `占 C 档 42.8% / 占总组合(ABC) 19.2%`，禁止模糊的 `仓位 42.8%`。

### 5.5 外部环境监控

#### 5.5.1 数据可信度规范（P0）

每条外部信号必须包含：

| 字段 | 说明 | 示例 |
|------|------|------|
| 来源名称 | 数据发布机构 | 国家统计局 |
| 来源链接 | 直接跳转到原始数据 | stats.gov.cn/xxx |
| 发布时间 | 数据原始发布日期 | 2026-05-30 09:30 |
| 抓取/更新时间 | 系统获取时间 | 2026-05-30 08:45 自动 |
| 可信等级 | A/B/C | A |
| 维护方式 | 自动抓取/人工录入 | 自动抓取 |

**可信等级**：

| 等级 | 标准 | 示例 | 是否入执行决策层 |
|------|------|------|------|
| A | 官方/交易所原始数据，可自动抓取 | PMI、LPR、LME 铝价、公司公告 | 是 |
| B | 第三方数据库/行业机构 | 理杏仁 PE、Wind 数据 | 是 |
| C | 媒体报道、研报观点、LLM 提炼 | 新闻摘要、分析师共识 | 仅参考 |
| — | 社区传闻、非正式消息 | — | 不入任何层 |

#### 5.5.2 信号影响结构化（P1）

每条宏观/行业信号按以下结构展开，逐标的分析：

```
PMI 5月官方制造业 49.5（前值 50.4，跌破荣枯线）           [事实]
  来源：国家统计局 | 链接 | 发布：5/30 09:30 | 可信度：A | 自动抓取

  对比亚迪 002594.SZ：短期偏利空↘                          [解读]
    逻辑：制造业+消费链条承压，汽车作为大件消费受冲击
    时间维度：1-4 周 | 置信度：中

  对中证A500 563360.SH：短期偏利空↘                        [解读]
    逻辑：制造业景气回落压制风险偏好，宽基指数承压
    时间维度：1-4 周 | 置信度：中

  对中证红利 515180.SH：中性偏多↗                          [解读]
    逻辑：风险偏好下降时，防御型高股息资产吸引力上升
    时间维度：1-4 周 | 置信度：中

  → 不改变今日任务。A500 定投节奏不变，红利建仓按计划执行。     [动作]
```

#### 5.5.3 三层信息结构标注

| 层 | 定义 | 标注 |
|------|------|------|
| [事实] | 原始数据，不含判断 | 无背景色 |
| [解读] | 对哪些资产有什么影响，逻辑是什么 | 灰色背景 |
| [动作] | 是否改变今日任务 | 黄/红边框 |

### 5.8 Thesis 维护（模板化）

每个 thesis 更新任务使用固定模板字段：

| 字段 | 说明 |
|------|------|
| 原始买入假设 | 当初为什么买入 |
| 当前验证状态 | 假设被验证 / 被削弱 / 被证伪 |
| 价格归因 | 市场因素 / 行业因素 / 公司因素 / 仓位管理 |
| 持有决策 | 继续持有 / 减仓 / 清仓 / 加仓 / 观察 |
| 新触发条件 | 什么情况下改变决策 |
| 下次复盘日期 | 不超过 30 天 |

```
📝 隆基绿能 601012.SH · T-002 · 截止 6/1（2天后）

原始假设：BC 电池技术领先，集中式电站龙头，产能出清后盈利修复
验证状态：⚠️ 部分被削弱
  · 被验证：BC 电池效率 28.13% 世界纪录，技术壁垒仍在
  · 被削弱：组件价格持续下行，产能出清慢于预期，毛利率承压
价格归因：行业因素（产能过剩）+ 公司因素（盈利低于预期）
持有决策：清仓（组合纪律驱动，非 thesis 完全证伪）
新触发条件：组件价格企稳回升 + 毛利率连续两季改善 → 重新评估
下次复盘：2026-06-30
```

### 5.9 LLM 摘要（增强）

三段式结构：

```

引用依据：PMI 49.5 · 碳酸锂 ¥9.8万 · 组件集采 · LME铝 $2,580 · A500 PE 97%

  偏空因素：PMI跌破荣枯线、组件价格下行、欧盟关税调查
  偏多因素：碳酸锂降价、铝价温和上涨
  置信度：中 | 最大不确定性：比亚迪 5 月销量今日待发布

今日任务影响逐条确认：
  ✅ 中证红利建仓(B-P1-02)：不受影响，按计划执行
```

## 6. 数据规范

### 6.1 标的标识

所有涉及交易、持仓、监控、告警、Thesis 的地方统一格式：

```
名称 + 代码 + 市场后缀

后缀：.SH（A股上海）/ .SZ（A股深圳）/ .HK（港股）/ .US（美股）

示例：
  比亚迪 002594.SZ
  隆基绿能 601012.SH
  理想汽车 02015.HK
  中证红利 ETF 515180.SH
```

### 6.2 金额估算

所有交易金额标注估算价格和价格日期：

```
格式：估 ¥XX,XXX（估 ¥XX.XX，YYYY-MM-DD 收盘）

示例：估 ¥54,144（估 ¥15.04，2026-05-29 收盘）
```

### 6.3 颜色纪律

| 颜色 | 用途 | 禁止用法 |
|------|------|------|
| 红 | 必须处理、已违规、止损触发 | 不用于普通下跌/亏损 |
| 黄 | 需要确认、待观察、有条件 | 不用于"今日关注"（用蓝色） |
| 蓝 | 信息提醒、数据标签 | — |
| 绿 | 已完成、条件满足、利好信号 | — |
| 灰 | 无动作、已失效、阻塞状态 | — |

### 6.4 仓位口径

所有百分比标注分母，禁止模糊写法：

```
✅ 占 C 档 42.8% / 占总组合(ABC) 19.2%
❌ 仓位 42.8%
```
