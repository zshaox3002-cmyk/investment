2.2 特殊资产处理说明

限制性股票（RSU）处理原则：RSU 等非常规资产不纳入三档架构，不参与日常再平衡计算。减持所得直接进入 A/B/C 再平衡流程。

以美团RSU为例，系统采用两阶段再平衡框架：

阶段时间处理方式
第一阶段当前 → 2027-11美团RSU 排除在再平衡基准之外；减持所得按 30%→A / 70%→B 分配
第二阶段2027-11 之后第二档完成后，RSU 资产重新纳入整体计算，重设再平衡基准

3. 每日战情室（Dashboard）

每日战情室是本系统的统一指挥中心。所有待办事项汇整到 config/execution_tracker.yaml，由 scripts/dashboard.py 生成自包含 HTML 页面，提供今日待办、执行进度、合规预警、盘前/盘后检查清单的一站式视图。

设计理念：数据层与视图层分离。execution_tracker.yaml 是唯一真相源，手动更新状态；DASHBOARD.html 由脚本自动生成，不手动编辑。支持三种模式切换：标准模式、盘前模式（--pre-market）、盘后模式（--post-market）。

自动化脚本

9 个核心 Python 脚本，覆盖从每日战情室、每日监控到季度诊断的完整工作流。所有脚本读取 config/rules.yaml 作为核心配置。

dashboard.py
每日盘前 09:00 / 盘后 15:15 运行。读取 execution_tracker.yaml 生成 DASHBOARD.html，统一展示今日待办、执行进度、合规预警、盘前/盘后检查清单。支持 --pre-market / --post-market 模式切换。
python scripts/dashboard.py python scripts/dashboard.py --pre-market python scripts/dashboard.py --post-market
→ DASHBOARD.html
daily_snapshot.py
每日 15:15 运行（A 股收盘后 15 分钟）。拉取日终行情（腾讯财经 API 批量请求），生成持仓快照，触发告警检查。持有港股时可推迟至 16:15。
python scripts/daily_snapshot.py
→ reviews/daily/YYYY-MM-DD.md
alert_monitor.py
告警触发后运行。调用 Claude 分析告警原因，生成处置建议，支持手机推送通知。
python scripts/alert_monitor.py
→ alerts/YYYY-MM-DD_*.md
weekly_brief.py
每周一 09:00 运行（开盘前 30 分钟）。汇总本周告警、持仓状态快照、近 14 天截止事项，生成周报框架。
python scripts/weekly_brief.py
→ reviews/weekly/YYYY-WNN.md
hermes_enrich.py
每周一 09:02 运行（weekly_brief.py 之后）。调用本地 Hermes Agent 搜索持仓公司本周公告/财报窗口和宏观数据发布日历，自动写入周报"本周关注事项"区域，无需手动填写。
python scripts/hermes_enrich.py
→ 追加写入 reviews/weekly/YYYY-WNN.md
monthly_update.py
月末最后一个交易日 15:30 运行。更新所有持仓的 thesis 评分卡，生成月度绩效报告。月度数据完整后执行，评分不受下月行情影响。
python scripts/monthly_update.py

quarterly_review.py

python scripts/rebalance_check.py
→ reviews/rebalance/YYYY-MM-DD.md
trade_log.py
交易冷静期管理。扫描 decision_*.md，显示哪些决策已过冷静期可执行，并创建执行日志。

6. AI Skills

5 个 Claude 提示词模板，存放于 prompts/ 目录。在 Claude Code 中通过 /skill-name 触发，或直接将模板内容粘贴给 Claude。

thesis-tracker
触发时机：每月末 / 财报后 / 回撤 ≥15% 时
对持仓股票进行五维度评分（业务质量、竞争优势、管理层、估值、风险），输出结构化评分卡。评分 ≤2.0 持续两季度触发强制清仓。
comps-analysis
触发时机：买入前 / 每季度必跑
对目标股票进行估值横向对比（PE/PB/PS/EV-EBITDA），与同行业可比公司对比，输出估值溢价/折价分析。
earnings-analysis
触发时机：财报发布后 48 小时内必跑
解读财报关键指标（营收、毛利率、经营现金流、资本开支），对比预期与实际，评估是否触发 thesis 更新。
ic-memo
触发时机：新建仓前必须先跑（硬性要求）
买入决策备忘录。强制回答七个反人性问题，输出结构化买入理由、仓位规划、退出预案。未跑 ic-memo 不得建仓。
idea-generation
触发时机：每月扫候选池
基于当前宏观判断（config/macro.md）和组合缺口，扫描候选标的，输出初步筛选列表供进一步研究。

7. 风控铁律

所有规则存储于 config/rules.yaml，修改需经 7 天冷静期 + 书面理由。在已存在 active_breaches 时，禁止提高阈值或添加自我豁免。

7.1 仓位集中度限制

规则阈值触发动作宽限期
单一个股上限25%强制降权，目标降至 ≤20%30 天
前 3 大持仓合计50%预警，触发组合复盘—
前 5 大持仓合计70%强制降仓60 天
单一行业上限40%预警—
新能源/电力链主题35%（硬上限 50%）预警 / 强制降仓—

7.2 组合回撤控制

级别阈值措施
L1 预警-10%组合复盘；补仓需额外满足 thesis ≥ 4.0
L2 控制-15%暂停新建个股 30 天；禁止向亏损个股补仓
L3 硬刹车-20%停止所有新建仓 60 天；主动个股总仓位降至 ≤25%；现金/短债提高到 ≥15%

7.3 单股止损线

级别浮亏阈值措施
L1 关注-10%记录下跌原因；补仓需 thesis ≥ 4.0
L2 审视-20%7 天内给出持有/减仓/清仓决策；强制更新 thesis
L3 强制-30%thesis < 3.0：减仓至少 50%；thesis ≥ 3.0：只维持不补仓

7.4 交易行为规则

规则参数
新建仓冷静期7 天
卖出决策冷静期3 天（强制减仓豁免）
补仓决策冷静期5 天
单笔交易上限总组合 5% / 该标的仓位 40%
单日交易总额上限总组合 10%
初始建仓上限总组合 5%

资金路由规则（减仓回笼资金）

A 档 30%
B 档 70%

减仓所得：30% 进 A 档（现金/短债），70% 进 B 档（核心ETF），0% 重新进入个股

7.5 规则修改约束

  • 修改 rules.yaml 需经 7 天冷静期 + 书面理由
  • 存在 active_breaches 时，禁止提高违规相关阈值
  • 禁止为亏损个股单独创建豁免
  • 禁止在亏损状态下提高单股仓位上限
  • 每季度回顾一次铁律合理性

7.6 参考案例：违规触发与整改流程

案例：单股超仓（南山铝业）
持仓占 C 仓位 45.81%，超单股上限 25%。触发强制降权告警，30 天宽限期内必须降至 ≤25%。整改路径：创建 decision_*.md → 冷静期 3 天 → 执行减仓 → 回笼资金按 30%/70% 分配至 A/B 档 → 更新持仓配置。

整改链路：告警触发 → 创建决策文件（trades/decision_*.md）→ 冷静期等待 → 执行交易（trade_log.py log）→ 更新配置（holdings.csv / core_etf.csv）→ 验证合规（daily_snapshot.py）

8.4 反人性检查清单

每次决策前必须自问：

  • 如果这笔钱现在是现金，今天的价格我会买它吗？
  • 我是在做决策，还是在为已有的情绪寻找理由？
