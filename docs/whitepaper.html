特殊资产处理说明

限制性股票（RSU）处理原则：RSU 等非常规资产不纳入三档架构，不参与日常再平衡计算。减持所得直接进入 A/B/C 再平衡流程。

以美团RSU为例，系统采用两阶段再平衡框架：

阶段时间处理方式
第一阶段当前 → 2027-11美团RSU 排除在再平衡基准之外；减持所得按 30%→A / 70%→B 分配
第二阶段2027-11 之后第二档完成后，RSU 资产重新纳入整体计算，重设再平衡基准

每日战情室（Dashboard）

每日战情室是本系统的统一指挥中心。由 inv dashboard render 从 SQLite 数据库读取持仓、告警、待执行清单、违规状态，生成自包含 HTML 页面，提供今日待办、执行进度、合规预警、盘前/盘后检查清单的一站式视图。

设计理念：数据层与视图层分离。SQLite 是唯一真相源；DASHBOARD.html 由 inv dashboard render 自动生成，不手动编辑。支持三种模式：标准模式、盘前模式（--pre-market）、盘后模式（--post-market）。

日常工作流

执行时间设计原则：所有时间点以 A 股收盘（15:00）为核心锚点。收盘后执行快照可确保日终价格已确定，告警不因盘中波动误触发；周一开盘前读简报可在市场噪音介入前完成信息摄入；月末/季末收盘后执行更新可保证数据完整性。

每日（5–10 分钟）· 盘前 09:00 + 盘后 15:15

A 股 09:30 开盘，盘前 09:00 运行 Dashboard 查看今日待办。15:00 收盘后等待 15 分钟让数据源完成日终结算后执行快照。若持有港股，可推迟至 16:15（港股 16:00 收盘）。

CLI 命令参考

所有操作通过 inv 统一入口执行。安装：pip install -e ".[dev]"，之后 inv --help 查看完整命令树。

inv snapshot pull
每日 15:15 运行。拉取日终行情（腾讯财经 API），写入 holdings + quotes 表，触发 8 项告警检查，去重写入 alerts 表。
inv snapshot pull
→ DB: quotes / holdings / alerts 更新　→ reviews/daily/YYYY-MM-DD.md
inv dashboard render
从 DB 读取持仓、告警、待执行清单、违规状态，生成 DASHBOARD.html。支持 --mode pre-market / post-market。
inv dashboard render inv dashboard render --mode pre-market inv dashboard render --mode post-market
→ DASHBOARD.html
inv exec monitor
检查所有 armed 状态的 stop_rules，对比最新行情，触发则写入 alerts 表并推送通知。不自动下单，保留人工拍板。
inv exec monitor
→ DB: alerts 更新（triggered stop_rules）
inv trade decision
交易决策管理。new 创建决策记录 + stub markdown；list 查看冷静期状态；show ID 查看详情。
inv trade decision new --code 600219 --type REDUCE inv trade decision list inv trade decision show 3
→ DB: decisions 表　→ trades/decision_NNN.md
inv trade log / apply
log 记录实际成交；apply 将成交反向更新 holdings 表，保持持仓数据与实际一致。
inv trade log --decision-id 3 --shares 6700 --price 5.41 inv trade apply 3
→ DB: trades / holdings 更新
inv trade stop
止损止盈规则管理。add 添加规则（GRID_SELL/GRID_BUY/HARD_DD）；list 查看所有 armed 规则及当前触发状态。
inv trade stop add --decision-id 3 --type GRID_SELL --price 5.81 inv trade stop list
→ DB: stop_rules 表
inv thesis
sync 同步 theses/ 目录的 frontmatter 到 DB；score 写入月度评分；stale 列出超过更新周期的 thesis。
inv thesis sync inv thesis score --code 600219 --score 3.27 inv thesis stale
→ DB: theses / thesis_scores 更新
inv candidate
scan 调用 akshare 扫描全市场候选池（hard/soft 过滤 + 合规检查）；list 查看候选池；promote 晋升为 IC Memo 研究对象。
inv candidate scan --quick inv candidate list inv candidate promote 5
→ DB: candidates 表
inv review
log 记录闭仓复盘（错误归因 12 种固定 enum）；stats 输出三月/六月错误频次统计，识别行为模式。
inv review log --trade-id 5 inv review stats
→ DB: trade_reviews / review_errors 更新
inv migrate
run 执行 01-08 迁移脚本（幂等）；verify 5 项数据对账，差异 > 0.01% 报错，输出 diff_report.md。
inv migrate run inv migrate verify
→ data/migration_diff_report.md
inv data
tables 列出 DB 所有表和视图；export csv 将 DB 反向导出为 CSV（灾难恢复 / 人工审阅）。
inv data tables inv data export csv
→ data/export/*.csv
v1 → v2 命令对照：原 python scripts/daily_snapshot.py → inv snapshot pull；python scripts/dashboard.py → inv dashboard render；python scripts/trade_log.py log → inv trade log；python scripts/monthly_update.py → inv thesis score。v1 scripts/ 目录保留 ≥90 天作对账兜底。

AI Skills

5 个 Claude 提示词模板，存放于 prompts/ 目录。在 Claude Code 中通过 /skill-name 触发，或直接将模板内容粘贴给 Claude。

thesis-tracker
触发时机：每月末 / 财报后 / 回撤 ≥15% 时
对持仓股票进行五维度评分（业务质量、竞争优势、管理层、估值、风险），输出结构化评分卡。评分 ≤2.0 持续两季度触发强制清仓。
comps-analysis
触发时机：买入前 / 每季度必跑
对目标股票进行估值横向对比（PE/PB/PS/EV-EBITDA），与同行业可比公司对比，输出估值溢价/折价分析。
earnings-analysis
触发时机：财报发布后 48 小时内必跑
解读财报关键指标（营收、毛利率、经营现金流、资本开支），对比预期与实际，评估是否触发 thesis 更新。
ic-memo
触发时机：新建仓前必须先跑（硬性要求）
买入决策备忘录。强制回答七个反人性问题，输出结构化买入理由、仓位规划、退出预案。未跑 ic-memo 不得建仓。
idea-generation
触发时机：每月扫候选池
基于当前宏观判断（config/macro.md）和组合缺口，扫描候选标的，输出初步筛选列表供进一步研究。

风控铁律

所有规则存储于 config/rules.yaml，修改需经 7 天冷静期 + 书面理由。在已存在 active_breaches 时，禁止提高阈值或添加自我豁免。

规则阈值触发动作宽限期
单一个股上限25%强制降权，目标降至 ≤20%30 天
前 3 大持仓合计50%预警，触发组合复盘—
前 5 大持仓合计70%强制降仓60 天
单一行业上限40%预警—
新能源/电力链主题35%（硬上限 50%）预警 / 强制降仓—

组合回撤控制

级别阈值措施
L1 预警-10%组合复盘；补仓需额外满足 thesis ≥ 4.0
L2 控制-15%暂停新建个股 30 天；禁止向亏损个股补仓
L3 硬刹车-20%停止所有新建仓 60 天；主动个股总仓位降至 ≤25%；现金/短债提高到 ≥15%

单股止损线

级别浮亏阈值措施
L1 关注-10%记录下跌原因；补仓需 thesis ≥ 4.0
L2 审视-20%7 天内给出持有/减仓/清仓决策；强制更新 thesis
L3 强制-30%thesis < 3.0：减仓至少 50%；thesis ≥ 3.0：只维持不补仓

规则修改约束

  • 修改 rules.yaml 需经 7 天冷静期 + 书面理由
  • 存在 active_breaches 时，禁止提高违规相关阈值
  • 禁止为亏损个股单独创建豁免
  • 禁止在亏损状态下提高单股仓位上限
  • 每季度回顾一次铁律合理性

参考案例：违规触发与整改流程

案例：单股超仓（南山铝业）
持仓占 C 仓位 45.81%，超单股上限 25%。触发强制降权告警，30 天宽限期内必须降至 ≤25%。整改路径：创建 decision_*.md → 冷静期 3 天 → 执行减仓 → 回笼资金按 30%/70% 分配至 A/B 档 → 更新持仓配置。

整改链路：告警触发 → 创建决策文件（trades/decision_*.md）→ 冷静期等待 → 执行交易（trade_log.py log）→ 更新配置（holdings.csv / core_etf.csv）→ 验证合规（daily_snapshot.py）

反人性检查清单

每次决策前必须自问：

  • 如果这笔钱现在是现金，今天的价格我会买它吗？
  • 我是在做决策，还是在为已有的情绪寻找理由？
  • 如果决策错了，我能承受的最大损失是多少？
  • 这个决策违反了哪一条 rules.yaml？我有正当豁免理由吗？
  • 30 天后回看这个决策，我会为它感到骄傲吗？
  • 我买它，是因为未来风险收益比好，还是因为我已经亏损不甘心？
  • 如果这只股票今天清仓，我会重新买回同样仓位吗？

事件驱动因果推理

v2.1 新增的因果推理子系统，将宏观事件、行业动态与持仓标的之间的因果关系建模为有向图，每日自动扫描新闻信号、更新节点激活度、评估影响链传导，最终在 Dashboard 中展示"影响链异动"。

核心理念：市场不是随机波动的集合，而是由事件驱动的因果网络。一只股票涨跌的背后，是地缘冲突→大宗商品→行业成本→公司利润的逐级传导。causal 模块将这些传导链显式化、可量化、可追溯。

因果图谱结构

图谱由节点（Node）和有向边（Edge）组成，按传导层级组织：

层级含义节点类型示例
L0_geopolitical地缘政治事件中东-军事冲突、中美贸易摩擦
L1_macro宏观变量WTI油价、美元指数DXY、中国PMI
L2_industry行业/板块有色金属-铝、能源-石油、新能源-光伏
L3_holding持仓标的600219-南山铝业、000858-五粮液

5 维边评分体系

每条因果边由 5 个维度综合评分，加权计算边强度：

strength = d1 × 0.30 + d2 × 0.25 + d3 × 0.25 + d4 × 0.10 + d5 × 0.10
维度含义范围权重
d1_directness直接性：A→B 因果关系是否直接1–530%
d2_elasticity弹性：A 变动 1%，B 变动多少1–525%
d3_consistency一致性：历史中该关系是否稳定1–525%
d4_speed速度：传导时滞多长1–510%
d5_uniqueness独特性：该传导链是否独立于其他因素1–510%

节点生命周期

每个节点维护一个激活分数（activation_score），根据新闻信号动态变化：

new_activation = max(0, old_activation × e−0.03×days + |signal_strength| × 20)

根据激活分数，节点在三种状态间迁移：

状态条件含义
activeactivation ≥ 5活跃，参与路径搜索和影响评估
dormantactivation < 5 持续 14 天休眠，暂停参与传播
archiveddormant 持续 90 天归档，需重新激活才能恢复

每日盘后运行 inv causal daily，自动执行三步流水线：

1
信号扫描
抓取新闻→去重→LLM 分类→写入 news_signals 表
inv causal scan
2

衰减旧节点激活度、迁移 dormant/archived 状态

3

路径影响计算

对每条从事件到持仓的传导路径，计算复合影响：

impact = signal_strength × Π(strengthi / 5) × sign(directioni) × 0.85(n−1)

其中 0.85 是每跳衰减系数（跳数越多，传导越弱）。多路径影响聚合后，按阈值分级：

等级阈值含义颜色
L5|impact| ≥ 0.50强影响，建议立即评估 thesis红色
L4|impact| ≥ 0.30中等影响，建议加入复盘清单黄色
L3|impact| ≥ 0.15弱影响，建议观察蓝色
L1–L2< 0.15可忽略，不写入 chain_assessments灰色

只有 L3 及以上的评估结果才会写入 chain_assessments 表并在 Dashboard 展示。

图谱发现与审批

因果图谱通过两种方式扩展：

LLM 自动发现

当持仓出现异常波动时，让 LLM 分析可能的事件→标的因果路径：

# 手动触发：指定事件和持仓
inv causal discover --code 600219 --event "中国PMI超预期回升"

# 自动扫描：检测近期波动 > 阈值的持仓
inv causal discover-auto --volatility 5.0 --lookback 3

LLM 会提出新的因果节点和边，以 pending_edges 形式暂存，需人工审批后才能加入图谱。

人工审批流程

1
查看待审批
列出所有 pending edges
inv causal review list
2
交互式审批
逐条 approve/reject/modify
inv causal review
3
单条操作
快速 approve/reject/modify 指定 ID
inv causal review approve 5

审批人可修改 5 维评分和方向，系统记录所有审批日志到 causal_review_log 表。

Dashboard 集成

评估结果直接渲染到 Dashboard 的"影响链异动"区块：

  • 按 |impact_score| 降序排列
  • 颜色编码：L5 红色 / L4 黄色 / L3 蓝色
  • 显示关键传导链节点序列、信号数量、建议动作
  • 点击行展开 LLM 生成的详细解读
  • 无活跃影响链时显示"今日无活跃影响链"空状态
# 每日盘后一键运行
inv causal daily
inv dashboard render && open DASHBOARD.html

CLI 命令参考

inv causal node add
添加因果节点，支持 6 种类型和 4 个层级。关键词可自动生成。
inv causal node add --name "中国PMI" --type macro --layer L1_macro
inv causal node list / show
列出节点（可按 layer/state/type 过滤）或查看单个节点详情。
inv causal node list --layer L3_holding inv causal node show "WTI油价"
inv causal edge add
手动添加有向边，指定 5 维评分和传导时滞。
inv causal edge add --from "WTI油价" --to "能源-石油" --d1 5 --d2 4 --d3 5 --d4 3 --d5 5
inv causal discover
通过 LLM 发现事件→持仓的因果路径，新节点和边写入 pending 表。
inv causal discover --code 600219 --event "中国PMI超预期回升" inv causal discover-auto --volatility 5.0
inv causal review
交互式审批 pending edges：approve/reject/modify/skip。支持单条命令行操作。
inv causal review inv causal review approve 5 inv causal review modify 3 --d1 4 --direction 1
inv causal scan
每日新闻信号扫描：抓取→去重→LLM 分类→写入信号表→更新节点激活度。
inv causal scan inv causal scan --dry-run
inv causal lifecycle
应用激活衰减和生命周期状态迁移。review 查看最近 N 天内状态变化的节点。
inv causal lifecycle update inv causal lifecycle review --days 90
inv causal assess
评估今日信号对持仓的影响：路径搜索→影响计算→LLM 解读→持久化 L3+ 结果。
inv causal assess inv causal assess --code 600219 --explain
inv causal graph
输出持仓的因果子图，支持 Mermaid 可视化或 JSON 格式。
inv causal graph --code 600219 --format mermaid --hops 3
inv causal daily
每日一键流水线：scan → lifecycle → assess，三步串联执行。
inv causal daily inv causal daily --dry-run
