## 一、数据文件体系

### 1.1 核心数据文件

| 文件名 | 位置 | 更新频率 | 维护方式 | 用途 |
|------|------|---------|---------|------|
| **portfolio_snapshot.csv** | config/ | 日度 | 自动 | 当前完整持仓快照 |
| **portfolio_history.csv** | data/ | 日度 | 自动 | 历史快照追踪 |
| **portfolio_transactions.csv** | data/ | 交易后 | 半自动 | 交易日志审计 |
| **holdings.csv** | config/ | 交易后 | 手动 | C 档个股清单 |
| **core_etf.csv** | config/ | 交易后 | 手动 | B 档 ETF 清单 |
| **cash_positions.csv** | config/ | 月度 | 手动 | A 档现金/短债 |

### 1.2 文件关系图

```
portfolio_snapshot.csv（当前快照）
    ↓
    ├─ 日度自动生成（daily_snapshot.py）
    ├─ 包含 A/B/C 三档完整数据
    └─ 用于日常监控

portfolio_history.csv（历史追踪）
    ↓
    ├─ 日度追加（daily_snapshot.py）
    ├─ 保留完整历史记录
    └─ 用于回溯分析

portfolio_transactions.csv（交易日志）
    ↓
    ├─ 交易后更新（trade_log.py）
    ├─ 记录每笔交易详情
    └─ 用于审计和复盘

holdings.csv / core_etf.csv / cash_positions.csv（配置清单）
    ↓
    ├─ 交易后手动更新
    ├─ 作为 portfolio_snapshot.csv 的数据源
    └─ 用于配置管理
```

## 二、更新频率与触发条件

### 2.1 日度更新（每日收盘后）

**触发条件**：每个交易日收盘后
**执行脚本**：`scripts/daily_snapshot.py`
**更新内容**：
- 读取 holdings.csv / core_etf.csv / cash_positions.csv
- 获取当日收盘价
- 计算当日市值、盈亏、仓位占比
- 生成 portfolio_snapshot.csv（覆盖）
- 追加 portfolio_history.csv（追加）
- 检查告警条件

**输出文件**：
- `config/portfolio_snapshot.csv`（当前快照）
- `data/portfolio_history.csv`（历史追踪）
- `reviews/daily/[YYYY-MM-DD].md`（日报）

**验证方式**：
- 总市值 = A 档 + B 档 + C 档
- 仓位占比 = 各档市值 / 总市值
- 盈亏 = 当前市值 - 成本价 × 股数

---

### 2.2 交易后更新（实时）

**触发条件**：每笔交易执行后
**执行脚本**：`scripts/trade_log.py`
**更新内容**：
- 更新 holdings.csv / core_etf.csv / cash_positions.csv
- 记录交易详情到 portfolio_transactions.csv
- 生成交易日志 `trades/log_*.md`

**更新步骤**：
1. 交易执行 → 获取成交价、成交量、成交时间
2. 更新对应的 CSV 文件（股数、成本价）
3. 记录到 portfolio_transactions.csv
4. 生成 trades/log_*.md 日志
5. 触发 daily_snapshot.py 重新计算

**验证方式**：
- 交易前后的股数是否一致
- 成本价是否正确摊薄
- 资金分配是否按规则执行

---

### 2.3 月度验证（每月末）

**触发条件**：每月最后一个交易日
**执行脚本**：`scripts/monthly_update.py`
**更新内容**：
- 完整验证 A/B/C 三档配置
- 更新 thesis 评分卡
- 检查是否需要再平衡
- 生成月度报告

**验证清单**：
- [ ] 总市值是否与各档之和一致？
- [ ] A/B/C 占比是否符合目标？
- [ ] 是否有未记录的交易？
- [ ] 成本价是否正确？
- [ ] 是否有数据异常？

**输出文件**：
- `reviews/monthly/[YYYY-MM].md`（月报）
- 更新后的 holdings.csv / core_etf.csv / cash_positions.csv

---

### 2.4 季度诊断（每季度末）

**触发条件**：每季度最后一个交易日
**执行脚本**：`scripts/quarterly_review.py`
**更新内容**：
- 完整的组合诊断
- Thesis 五支柱评分更新
- 集中度分析
- 再平衡计划

**输出文件**：
- `reviews/quarterly/[YYYY-Q].md`（季报）

## 三、数据维护流程

### 3.1 日常维护流程（每日）

```
收盘后（16:00）
    ↓
运行 daily_snapshot.py
    ├─ 读取 holdings.csv / core_etf.csv / cash_positions.csv
    ├─ 获取当日收盘价（从 API 或手动输入）
    ├─ 计算市值、盈亏、仓位占比
    ├─ 生成 portfolio_snapshot.csv
    ├─ 追加 portfolio_history.csv
    ├─ 检查告警条件
    └─ 生成 reviews/daily/[YYYY-MM-DD].md
    ↓
5-10 分钟查看快照 + 告警
    ↓
记录异常或需要处理的事项
```

**所需时间**：5-10 分钟（自动化 + 人工审查）

---

### 3.2 交易维护流程（交易后）

```
交易执行
    ↓
记录交易详情（代码、股数、价格、时间）
    ↓
更新对应 CSV 文件
    ├─ holdings.csv（C 档个股）
    ├─ core_etf.csv（B 档 ETF）
    └─ cash_positions.csv（A 档现金）
    ↓
运行 trade_log.py
    ├─ 记录到 portfolio_transactions.csv
    ├─ 生成 trades/log_*.md
    └─ 触发 daily_snapshot.py 重新计算
    ↓
验证交易是否正确记录
```

**所需时间**：5-10 分钟（手动 + 自动化）

---

### 3.3 月度验证流程（每月末）

```
月末最后一个交易日
    ↓
运行 monthly_update.py
    ├─ 完整验证 A/B/C 配置
    ├─ 检查是否有未记录的交易
    ├─ 验证成本价是否正确
    ├─ 更新 thesis 评分卡
    └─ 生成 reviews/monthly/[YYYY-MM].md
    ↓
30 分钟人工审查
    ├─ 检查数据异常
    ├─ 验证计算是否正确
    └─ 确认是否需要调整
    ↓
如有异常，修正并重新运行
```

**所需时间**：30-60 分钟（自动化 + 人工审查）

---

### 3.4 季度诊断流程（每季度末）

```
季度末最后一个交易日
    ↓
运行 quarterly_review.py
    ├─ 完整的组合诊断

    ├─ 集中度分析
    ├─ 再平衡计划
```

## 五、数据验证与异常处理

### 5.1 日度验证

**验证公式**：
```
总市值 = A 档市值 + B 档市值 + C 档市值

A 档市值 = 现金 + 货币基金 + 短债基金
B 档市值 = Σ(ETF 份额 × 当日净值)
C 档市值 = Σ(个股股数 × 当日收盘价)

总仓位占比 = 100%
A 档占比 = A 档市值 / 总市值
B 档占比 = B 档市值 / 总市值
C 档占比 = C 档市值 / 总市值
```

**异常判断**：
- 总市值偏差 > 1% → 检查数据源
- 仓位占比偏差 > 2% → 检查是否有未记录交易
- 成本价异常 → 检查是否有计算错误

### 5.2 异常处理流程

```
发现异常
    ↓
确认异常类型
    ├─ 数据输入错误 → 修正输入
    ├─ 计算错误 → 修正公式
    ├─ 未记录交易 → 补充记录
    └─ 系统错误 → 调试脚本
    ↓
修正后重新运行
    ↓
验证异常是否消除
    ↓
记录异常和处理过程
```
