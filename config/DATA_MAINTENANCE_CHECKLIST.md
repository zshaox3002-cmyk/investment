## 三、数据维护流程

### 日度流程（每日收盘后）

```
16:00 收盘
  ↓
运行 daily_snapshot.py
  ├─ 读取 holdings.csv / core_etf.csv / cash_positions.csv
  ├─ 获取当日收盘价
  ├─ 计算市值、盈亏、仓位占比
  ├─ 生成 portfolio_snapshot.csv
  ├─ 追加 portfolio_history.csv
  └─ 检查告警条件
  ↓
5-10 分钟查看快照 + 告警
  ↓
记录异常或需要处理的事项
```

### 交易流程（交易后）

```
交易执行
  ↓
记录交易详情
  ↓
更新 CSV 文件
  ├─ holdings.csv（C 档个股）
  ├─ core_etf.csv（B 档 ETF）
  └─ cash_positions.csv（A 档现金）
  ↓
运行 trade_log.py
  ├─ 记录到 portfolio_transactions.csv
  ├─ 生成 trades/log_*.md
  └─ 触发 daily_snapshot.py 重新计算
  ↓
验证交易是否正确记录
```

### 月度流程（每月末）

```
月末最后一个交易日
  ↓
运行 monthly_update.py
  ├─ 完整验证 A/B/C 配置
  ├─ 检查是否有未记录的交易
  ├─ 验证成本价是否正确
  ├─ 更新 thesis 评分卡
  └─ 生成 reviews/monthly/[YYYY-MM].md
  ↓
30 分钟人工审查
  ├─ 检查数据异常
  ├─ 验证计算是否正确
  └─ 确认是否需要调整
  ↓
如有异常，修正并重新运行
```

## 四、数据验证公式

### 总市值验证

```
总市值 = A 档市值 + B 档市值 + C 档市值

A 档市值 = 现金 + 货币基金 + 短债基金
B 档市值 = Σ(ETF 份额 × 当日净值)
C 档市值 = Σ(个股股数 × 当日收盘价)

总仓位占比 = 100%
A 档占比 = A 档市值 / 总市值
B 档占比 = B 档市值 / 总市值
C 档占比 = C 档市值 / 总市值
```

### 异常判断标准

- 总市值偏差 > 1% → 检查数据源
- 仓位占比偏差 > 2% → 检查是否有未记录交易
- 成本价异常 → 检查是否有计算错误

## 六、异常处理流程

```
发现异常
  ↓
确认异常类型
  ├─ 数据输入错误 → 修正输入
  ├─ 计算错误 → 修正公式
  ├─ 未记录交易 → 补充记录
  └─ 系统错误 → 调试脚本
  ↓
修正后重新运行
  ↓
验证异常是否消除
  ↓
记录异常和处理过程
```
