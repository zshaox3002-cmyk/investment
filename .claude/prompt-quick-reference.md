## 1️⃣ IC Memo — 买入决策备忘录

**何时使用**：任何新买入前（强制）  
**冷静期**：7 天  
**输出文件**：`trades/decision_*.md`

**核心步骤**：
1. 五支柱论证（参考 600219_thesis.md 的结构）
2. 三情景目标价计算（悲观 30% / 基础 50% / 乐观 20%）
3. 加权期望回报计算（门槛 ≥15%）
4. 估值三角形验证（PE/PB/分红折现三方法收敛）
5. 风险回报比评估
6. 仓位建议（简化 Kelly 公式）
7. 合规检查（7 项）+ 反人性检查（7 个问题）

**关键输出**：
- 三情景目标价 + 加权期望回报
- 仓位建议 + 建仓节奏
- 合规检查清单 + 反人性检查答案

**禁止事项**：
- ❌ 不接受"我看好这只股票"作为理由
- ❌ 不接受"摊低成本"作为补仓理由
- ❌ 不接受任何 rules.yaml 的豁免申请

## 2️⃣ Earnings Analysis — 财报解读

**何时使用**：财报发布后 48 小时内（强制）  
**输出文件**：更新 `theses/{code}_thesis.md` 的评分卡

**核心步骤**：
1. 关键数字提取（营收/利润/现金流/毛利率/指引）
2. 三维对比（同比/环比/vs 预期）
3. 利润质量评估（经营性现金流 vs 净利润）
4. 逐支柱验证（✅/⚠️/🔴）
5. 综合评分卡更新
6. 后续行动决策树

**关键输出**：
- 关键数字三维对比表
- 逐支柱验证结论（✅/⚠️/🔴）
- 新综合评分 + 评分变化
- 后续行动建议（维持/减仓/清仓/补仓）

**A 股特有关注**：
- 大股东减持计划
- 股权激励行权条件
- 分红方案（分红率、派息时间）

## 3️⃣ Thesis Tracker — 持仓论点跟踪

**何时使用**：每月末（强制）  
**输出文件**：更新 `theses/{code}_thesis.md` 的评分卡 + 版本日志

**核心步骤**：
1. 五维度评分标准明确（基本面/估值/技术面/催化剂/风险）
2. 支柱状态交通灯标记（✅/🟡/⚠️/🔴）
3. 证伪条件逐一核查
4. 综合评分重算（五维度加权平均）
5. 决策触发条件判断

**关键输出**：
- 五维度评分表（每维度 0-5 分）
- 支柱状态交通灯（✅/🟡/⚠️/🔴）
- 综合评分 + 环比变化
- 决策触发条件（维持/减仓/清仓/补仓）

**评分锚点**：
- 5 分：超预期改善 / 低估 / 向上 / 明确临近 / 极低
- 3 分：符合预期 / 合理 / 震荡 / 潜在 / 可控
- 1 分：低于预期 / 高估 / 向下 / 缺乏 / 很大

## 4️⃣ Comps Analysis — 估值横向对比

**何时使用**：买入前（强制）/ 季度必跑  
**输出文件**：用于 ic-memo 或 thesis.md 的估值章节

**核心步骤**：
1. 可比公司选择（3-5 家，同行业/市值相近/业务相似）
2. 核心估值指标提取（PE/PB/PS/EV-EBITDA + A 股指标）
3. 历史估值百分位计算（近 3/5 年）
4. 相对估值结论（低估/合理/高估 + 折溢价幅度）
5. 行业特殊调整说明

**关键输出**：
- 可比公司估值指标对比表
- 历史估值百分位表
- 相对折溢价分析
- 估值结论（低估/合理/高估）

**A 股特有指标**：
- 股息率（Dividend Yield）
- PB 分位数（近 3/5 年）
- ROE / ROIC
- 净利润增速（YoY）
- 自由现金流收益率（FCF Yield）

## 5️⃣ Idea Generation — 候选池扫描

**何时使用**：每月末（强制）  
**输出文件**：`reviews/monthly/[YYYY-MM]_idea_generation.md`

**核心步骤**：
1. 量化筛选门槛（PE < 25 / ROE > 10% / 股息率 > 2% / 市值 > 50 亿 / 无违规）
2. rules.yaml 合规前置检查（行业/主题集中度不超标）
3. 初步 Thesis 质量门槛（必须能写出 ≥3 个可验证支柱）
4. 结构化输出模板（每个候选标的固定字段）
5. 优先级排序（基于综合评分）

**关键输出**：
- 筛选结果统计（第一层/第二层/合规/Thesis 质量/最终候选）
- 候选清单（按优先级排序）
- 每个候选的基本信息 + 三支柱 + 风险 + 催化剂 + 估值 + 互补性

**A 股特有筛选**：
- 北向资金持仓趋势（机构认可度）
- 机构持仓变化（专业投资者看好）
- 解禁压力评估（未来 6 个月解禁量）

## 🔄 工作流集成

### 新建仓流程
```
idea-generation → comps-analysis → ic-memo → 买入（冷静期 7 天后）
```

### 持仓管理流程
```
每月末：thesis-tracker（更新评分卡）
财报后 48h：earnings-analysis（支柱验证）
根据结果：决定是否减仓/清仓/补仓
```

### 季度复盘流程
```
thesis-tracker（季度强制更新）
→ comps-analysis（季度估值更新）
→ 根据结果调整持仓
```
