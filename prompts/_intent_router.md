# 意图路由表 — Intent Router

> 本文件是"用户说人话 → 路由到正确 Skill"的中枢。
> 每个 Skill 定义：触发关键词、触发时机、典型用户口语示例（≥5 条）。
> 路由优先级：精确匹配 > 关键词匹配 > 模糊匹配。多 Skill 同时命中时，取优先级最高者。

---

## Skill 总览

| Skill ID | 名称 | 优先级 | 触发时机 |
|----------|------|--------|---------|
| `onboarding` | 目标与资产录入 | P0 | 首次使用 / 重置目标 |
| `position` | 仓位管理与再平衡巡检 | P0 | 每日盘后 / 主动查询 |
| `stock_screen` | 对话式选股 | P1 | 寻找新标的 |
| `risk` | 组合风险量化 | P1 | 风险评估 / 月度复盘 |
| `calendar` | 投资日历与催办 | P1 | 任务管理 / 提醒 |
| `attribution` | 业绩归因 | P1 | 复盘 / 季度总结 |
| `causal_insight` | 外部信息与因果归因 | P2 | 异动解释 / 新闻解读 |
| `cost` | 交易成本计算 | P2 | 买卖前估算 |
| `behavior` | 行为约束与决策日志 | P2 | 决策前检查 / 复盘 |

## Skill ① onboarding — 目标与资产录入

**触发关键词**：
`开始` `入门` `初始化` `设置目标` `录入资产` `我要开始` `第一次` `新手` `怎么用` `配置` `我的目标` `年化` `期望收益` `风险承受` `可投资金` `资产录入` `重新设置` `更新目标`

**触发时机**：
- 用户首次使用系统（无 user_profile 记录）
- 用户主动更新投资目标或风险偏好
- 用户询问如何开始使用

**典型用户口语示例**：
1. "我刚开始用，不知道怎么设置"
2. "帮我录入一下我的资产，大概有 50 万可以投"
3. "我想把目标年化收益改成 12%"
4. "我能承受的最大亏损是 20%，帮我配置一下"
5. "我有 30 万在 A 股，20 万在货币基金，怎么分配比较好"
6. "重新设置一下我的投资目标"
7. "我的风险承受能力比较低，帮我调整配置"
8. "我想 5 年后攒够 100 万，现在有 60 万，怎么规划"

**不触发条件**：用户询问具体持仓操作（→ `position`）、选股（→ `stock_screen`）

## Skill ② position — 仓位管理与再平衡巡检

**触发关键词**：
`仓位` `持仓` `巡检` `再平衡` `rebalance` `超仓` `偏离` `告警` `风控` `铁律` `回撤` `止损` `止盈` `减仓` `加仓` `清仓` `建仓` `我的仓位` `有没有问题` `风险` `超限` `配置偏离` `盘后` `今天怎么样` `组合状态`

**触发时机**：
- 每日盘后主动巡检
- 用户询问当前持仓状态
- 触发风控铁律时
- 单股回撤 ≥15% 时

**典型用户口语示例**：
1. "我的仓位有没有问题"
2. "今天盘后帮我看看"
3. "有没有超仓的情况"
4. "600519 跌了很多，需要减仓吗"
5. "帮我检查一下是否需要再平衡"
6. "我的 C 档占比是不是太高了"
7. "有没有触发什么风控规则"
8. "告警里有什么需要处理的"
9. "我的组合现在偏离目标多少"

**不触发条件**：用户询问风险量化指标（→ `risk`）、具体选股（→ `stock_screen`）

## Skill ③ stock_screen — 对话式选股

**触发关键词**：
`选股` `找股票` `候选` `扫描` `筛选` `推荐` `有什么好股` `低估值` `高股息` `成长股` `价值股` `什么股票值得买` `帮我找` `符合条件` `PE` `ROE` `股息率` `市值` `行业` `赛道` `白马` `蓝筹` `小盘` `大盘`

**触发时机**：
- 用户寻找新的投资标的
- 月度候选池扫描
- 用户描述选股条件

**典型用户口语示例**：
1. "帮我找一些低 PE 高股息的股票"
2. "有没有 ROE 超过 15% 的消费股"
3. "我想找一些市值 100 亿以下的成长股"
4. "最近有什么值得关注的新能源标的"
5. "帮我扫描一下候选池，看看有没有新机会"
6. "找一些类似茅台的高护城河消费股"
7. "PE 低于 15 倍、股息率超过 3% 的银行股有哪些"
8. "帮我筛选一下符合我选股标准的候选"

**不触发条件**：用户询问已持仓股票（→ `position`）、买入决策（→ 触发 `/ic-memo`）

## Skill ④ causal_insight — 外部信息与因果归因

**触发关键词**：
`为什么跌` `为什么涨` `原因` `新闻` `消息` `影响` `因果` `利好` `利空` `政策` `宏观` `行业` `事件` `异动` `解释` `分析一下` `最近发生了什么` `对持仓有什么影响` `信号` `因果链` `归因`

**触发时机**：
- 持仓出现异动（相对基准超额波动）
- 用户询问市场事件对持仓的影响
- 每日因果推理流水线

**典型用户口语示例**：
1. "600519 今天为什么跌了 5%"
2. "最近美联储加息对我的持仓有什么影响"
3. "帮我分析一下今天的市场新闻"
4. "新能源政策调整会影响我持有的比亚迪吗"
5. "我的组合今天跑输大盘，是什么原因"
6. "最近有什么宏观事件需要关注"
7. "帮我看看今天有没有影响持仓的重要消息"
8. "这个利空消息对我的仓位影响有多大"

**不触发条件**：用户询问业绩归因（→ `attribution`）、纯行情查询（→ `position`）

## 路由规则

### 优先级规则

1. **精确命令匹配**：用户输入包含 `/skill-name` 格式，直接路由，不走关键词匹配
2. **多关键词命中**：命中关键词最多的 Skill 优先
3. **同分平局**：按优先级（P0 > P1 > P2）决定
4. **无匹配**：返回 `null`，提示用户澄清意图

### 上下文修正规则

- 若当前对话已在某 Skill 流程中，优先保持上下文，不轻易切换
- 用户说"算了" / "换个话题" / "回到主菜单"时，重置上下文

### 歧义处理

当两个 Skill 得分相近（差距 ≤1 个关键词）时，输出澄清问题：
> "你是想 [Skill A 的功能]，还是 [Skill B 的功能]？"
