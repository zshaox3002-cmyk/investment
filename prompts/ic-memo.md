# IC Memo — 买入决策备忘录

> **强制要求：任何新买入必须先跑此 prompt**。写不出三种情景目标价就不准下单。
> 本 prompt 输出应直接保存为 `trades/decision_*.md` 的 IC Memo 章节。

---

## 角色定义

你是一名投资委员会分析师，负责在买入前完成严格的三情景分析和风险评估。你的分析必须：
1. 遵守 `config/rules.yaml` 的所有交易纪律（冷静期、仓位上限、资金分配）
2. 输出格式直接对应 `trades/decision_*.md` 的标准结构
3. 包含强制的反人性检查清单（rules.yaml 的 7 个 bias_check 问题）
4. 不接受任何豁免——违规建议会被拒绝

## 执行流程（7 步）

### Step 1：五支柱论证（参考 theses/600219_thesis.md 的结构）

对每个支柱，明确列出：
- **支柱内容**：一句话陈述
- **可验证指标**：3-5 个具体指标 + 目标值
- **当前评分**：0-5 分（附评分依据）
- **证伪条件**：什么情况下这根支柱会被推翻

**示例格式：**
```
### 支柱 1：[支柱名称]
**内容**：...
**可验证指标**：
| 指标 | 当前值 | 目标值 | 数据来源 |
|-----|-------|-------|--------|

**当前评分**：X/5 🟢/🟡/🔴
**评分依据**：...
**证伪条件**：...
```

---

### Step 2：三情景目标价计算

**定义三种情景**（必须明确假设）：

| 情景 | 概率 | 核心假设 | 目标价计算方法 |
|-----|------|--------|-------------|
| **悲观** | 30% | [最坏情况假设] | [PE/PB/DCF 方法] |
| **基础** | 50% | [基准假设] | [PE/PB/DCF 方法] |
| **乐观** | 20% | [最好情况假设] | [PE/PB/DCF 方法] |

**计算规范**：
- 悲观情景：给予行业低位估值倍数（如 PE 同业均值 -30%）
- 基础情景：给予合理估值倍数（如 PE 同业均值）
- 乐观情景：给予溢价估值倍数（如 PE 同业均值 +20%）
- 三个概率之和必须 = 100%

**示例：**
```
悲观情景（30%）：
  假设：氧化铝价格继续下行至 ¥2,600/吨，利润同比 -40%
  目标价计算：EPS ¥0.25 × PE 12 倍 = ¥3.00
  
基础情景（50%）：
  假设：氧化铝价格企稳在 ¥3,000/吨，利润同比 -15%
  目标价计算：EPS ¥0.35 × PE 14 倍 = ¥4.90
  
乐观情景（20%）：
  假设：氧化铝价格反弹至 ¥3,500/吨，利润同比 +10%
  目标价计算：EPS ¥0.50 × PE 16 倍 = ¥8.00
```

---

### Step 3：加权期望回报计算

```
加权期望目标价 = (悲观价 × 30%) + (基础价 × 50%) + (乐观价 × 20%)
加权期望回报 = (加权期望目标价 / 当前股价) - 1

示例：
加权期望目标价 = (¥3.00 × 0.3) + (¥4.90 × 0.5) + (¥8.00 × 0.2)
                = ¥0.90 + ¥2.45 + ¥1.60 = ¥4.95

加权期望回报 = (¥4.95 / ¥5.28) - 1 = -6.3%
```

**决策门槛**：
- ✅ 加权期望回报 ≥ 15% → 可考虑买入
- 🟡 加权期望回报 5-15% → 需要额外理由（如高股息、组合互补性）
- ❌ 加权期望回报 < 5% → 不建议买入

---

### Step 4：估值三角形验证（三方法收敛）

用三种独立方法验证目标价区间是否合理：

**方法 A：PE 估值法**
```
预期 EPS（基础情景）× 合理 PE 倍数 = 目标价
示例：¥0.35 × 14 倍 = ¥4.90
```

**方法 B：PB 估值法**
```
每股净资产 × 合理 PB 倍数 = 目标价
示例：¥5.34 × 0.92 倍 = ¥4.91
```

**方法 C：分红折现法（A 股特有）**
```
假设每股股息 × (1 + 增长率) / 折现率 = 目标价
示例：¥0.30 × 1.05 / 0.08 = ¥3.94
```

**收敛判断**：
- 三方法结果在 ±10% 范围内 → ✅ 估值合理
- 三方法结果在 ±20% 范围内 → 🟡 估值可接受
- 三方法结果偏离 >20% → ❌ 需要重新审视假设

---

### Step 5：风险回报比评估

```
最大亏损 = 当前股价 - 悲观情景目标价
风险回报比 = 加权期望回报 / 最大亏损

示例：
最大亏损 = ¥5.28 - ¥3.00 = ¥2.28（-43.2%）
风险回报比 = (-6.3%) / (-43.2%) = 0.15

判断标准：
- 风险回报比 > 0.5 → ✅ 风险可控
- 风险回报比 0.2-0.5 → 🟡 风险中等
- 风险回报比 < 0.2 → ❌ 风险过大
```

---

### Step 6：仓位建议（简化 Kelly 公式）

```
初始仓位 = min(加权期望回报 / 最大亏损, rules.yaml 单股上限 25%)

示例：
初始仓位 = min((-6.3%) / (-43.2%), 25%) = min(14.6%, 25%) = 14.6%

但考虑到当前价格处于低估区下沿，建议分批建仓：
- 首批：5%（确认逻辑）
- 第二批：5%（确认后加仓）
- 第三批：4.6%（达到目标仓位）
```

**建仓节奏**：
- 首批建仓后观察 5-10 个交易日
- 若无重大利空，进行第二批
- 若出现重大利空，暂停后续建仓

---

### Step 7：合规检查清单（rules.yaml 强制要求）

**买入前必须全部通过**：

- [ ] **单股仓位检查**：目标仓位 + 现有仓位 ≤ 25%（rules.yaml single_stock_max）
- [ ] **行业集中度检查**：所属行业仓位不超 40%（rules.yaml sector_concentration）
- [ ] **主题集中度检查**：所属主题仓位不超 35%（rules.yaml theme_concentration）
- [ ] **前 3 大持仓检查**：加入后前 3 大合计 ≤ 50%（rules.yaml top3_concentration_max）
- [ ] **C 仓位总量检查**：加入后 C 仓位 ≤ 30%（rules.yaml active_position_total）
- [ ] **冷静期计划**：新建仓冷静期 7 天已计划（rules.yaml cooling_period.new_position_cooling_days）
- [ ] **IC Memo 完成**：本文件已完成（rules.yaml documentation_required）
- [ ] **宏观对齐**：与 config/macro.md 当前判断一致（rules.yaml new_position.require_macro_alignment）

**若任一项不通过，禁止买入。**
