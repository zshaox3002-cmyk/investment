# 话术规范 — Voice Guide

> 所有 Skill 的用户可见输出必须遵守本规范。
> 核心原则：**技术编码只留底层，呈现层说人话，每条结论后必须附"所以你该做什么"。**

---

## 二、"所以你该做什么"模板

每条结论后必须附一句行动指引，格式固定：

```
所以你该做什么：[具体、可执行的下一步操作]
```

### 2.1 告警类

```
⚠️ [持仓名称] 触发了[规则人话名称]。
原因：[一句话解释为什么这条规则保护你]。
所以你该做什么：[具体操作，如"在 3 天冷静期内考虑是否减仓，减仓比例建议不超过 40%"]
```

### 2.2 风险类

```
📊 你的组合[风险描述，如"波动率为 18%，高于同类基准的 12%"]。
这意味着：[一句话解释对你的影响]。
所以你该做什么：[具体操作，如"考虑降低高波动持仓的比例，或增加低相关性资产"]
```

### 2.3 机会类

```
💡 [标的名称] 符合你的[筛选条件]。
关键数据：[2-3 个核心指标]。
所以你该做什么：[具体操作，如"运行 /ic-memo 做深度分析，通过后才能建仓"]
```

### 2.4 归因类

```
📈 这段时间你的收益为 [X%]，基准为 [Y%]，超额 [Z%]。
收益来源：[择时/选股/配置各贡献多少]。
所以你该做什么：[具体操作，如"选股贡献为负，建议复盘哪些持仓拖累了业绩"]
```

### 2.5 因果类

```
🔍 [事件描述] 对你的 [持仓名称] 有[影响程度]影响。
传导路径：[一句话描述因果链]。
所以你该做什么：[具体操作，如"关注后续政策落地情况，若确认利空则触发减仓决策"]
```

### 2.6 行为类

```
🧠 检测到[行为偏差名称]迹象。
表现：[具体描述]。
所以你该做什么：[具体操作，如"等待 24 小时冷静期，重新用 IC Memo 框架评估这笔交易"]
```

## 三、禁止事项

以下内容**禁止出现在用户可见输出中**：

- ❌ 原始技术编码（`C-006`、`stop_rule #17`、`d1..d5`、`chain_assessment_id`）
- ❌ 数据库 ID（`instrument_id=42`、`decision_id=7`）
- ❌ SQL 语句或技术堆栈信息
- ❌ 没有行动指引的结论（"你的仓位偏高"后面必须跟"所以你该做什么"）
- ❌ 模糊的行动指引（"你可以考虑操作"——必须说清楚做什么、什么时候做）

---

## 四、语气规范

- **直接**：不用"可能"、"也许"、"建议您考虑"——说清楚该做什么
- **简洁**：每条结论不超过 3 句话 + 1 句行动指引
- **有温度**：承认市场的不确定性，不假装全知全能
- **诚实**：数据不支持的结论不说，不确定的事情说"数据不足，建议人工判断"

---

## 五、输出结构模板

每个 Skill 的最终输出遵循以下结构：

```markdown
## [Skill 名称] — [日期/时间]

### 核心结论
[1-3 条最重要的结论，每条附行动指引]

### 详细数据
[支撑结论的数据，用表格或列表]

### 下一步行动清单
- [ ] [具体操作 1]（优先级：高/中/低）
- [ ] [具体操作 2]
```
